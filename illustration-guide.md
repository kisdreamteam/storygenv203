# Illustration Guide

Version: 1.1

Purpose:

This document defines how illustration prompts are written for the Nina & Nino series in V1.

V1 delivers **illustration prompts only** — no in-app image generation. Prompts must be consistent, copy-ready, and aligned with [character-bible.md](character-bible.md).

Authority: Below [character-bible.md](character-bible.md).

---

# 1. V1 Illustration Model

## What V1 provides

* One illustration prompt per story page (12 prompts per story)
* Prompts displayed in the UI with a copy button
* Prompts teachers can paste into external image tools (e.g., Midjourney, DALL·E, Ideogram, manual illustration briefs)
* Locked character descriptors and a standardized illustration continuity suffix on every prompt

## What V1 does NOT provide

* In-app image rendering
* Automatic image upload or storage
* Image editing inside the product
* Teacher-editable character profiles

Rule: **A good prompt teachers can use elsewhere is a successful V1 illustration output.**

---

# 2. Visual Style (Locked for V1)

## Overall look

* Warm, child-friendly, educational illustration style
* Soft flat or gentle painterly digital illustration (not photorealistic)
* Bright but not neon palette
* Clean uncluttered compositions with one clear focal action per page
* Safe, welcoming environments suitable for ages 4–6
* Consistent character appearance across all stories (per locked character bible)

---

# 3. Illustration Continuity System (Locked for V1)

This section defines the official V1 illustration continuity system. Every illustration prompt generated by the system must include the **locked visual suffix** below. This is the official V1 illustration framing standard.

## Required global suffix

Append this suffix to every prompt (after scene, character, setting, and mood details):

> 16:9 aspect ratio, landscape composition, zoomed-out view, full-body characters visible, extra empty space suitable for adding educational text later, no speech bubbles, no text, no labels, no watermarks, child-friendly educational illustration style, warm soft colors, simple shapes, friendly expressions, consistent character appearance, ages 4-6

## V1 illustration framing standard

| Rule | Requirement |
|------|-------------|
| Aspect ratio | 16:9 |
| Composition | Landscape |
| Camera | Zoomed-out view |
| Characters | Full-body visible when characters are on the page |
| Text space | Extra empty space reserved for future educational text overlays |
| Text in image | None — no speech bubbles, labels, words, or watermarks |
| Style | Child-friendly educational illustration |
| Characters | Locked appearances from [character-bible.md](character-bible.md) section 14 |

## Why this standard exists

* Keeps Nina, Nino, and family visually identical across stories
* Produces copy-ready prompts that work in external 16:9 image tools
* Leaves room for teachers to add vocabulary or lesson text outside the app
* Reduces illustration drift between pages and between stories

---

# 4. Character Consistency Rules

Every prompt that includes a character must use descriptors from [character-bible.md](character-bible.md) section 14.

Rules:

* **Official locked characters** (Nina, Nino, Mom, Dad, Grandpa, Ms. Lee) use their locked appearance descriptors on every appearance
* Mom always wears the yellow áo dài in prompts
* Tier 2 characters use suggested descriptors when present
* Tier 3 / teacher-introduced characters get a one-line stable descriptor when introduced; they are not persisted as profiles
* Do not invent new permanent visual traits for official locked characters
* Biscuit appears only when the story mentions Biscuit

If a character is not on the page:

* Do not include them in the prompt
* Focus on setting, objects, or remaining characters only

---

# 5. Per-Page Prompt Format

Each illustration prompt should follow this structure:

```
[Scene action from page text]. [Characters present with locked bible descriptors]. [Setting details]. [Mood/lighting]. 16:9 aspect ratio, landscape composition, zoomed-out view, full-body characters visible, extra empty space suitable for adding educational text later, no speech bubbles, no text, no labels, no watermarks, child-friendly educational illustration style, warm soft colors, simple shapes, friendly expressions, consistent character appearance, ages 4-6.
```

## Template fields

| Field | Required | Notes |
|-------|----------|-------|
| Scene action | Yes | One clear moment from the page text |
| Characters | If present | Use locked bible quick-reference descriptors |
| Setting | Yes | Home, classroom, park, etc. — match story |
| Mood/lighting | Yes | e.g., sunny morning, cozy evening, gentle rain |
| Continuity suffix | Yes | Locked suffix from section 3 |

## Example (page with Nina and Nino at the park)

```
Nina and Nino sit on a wooden bench feeding breadcrumbs to ducks at a small pond. Nina is a 6-year-old girl with medium skin tone, dark brown hair in two neat pigtails, bright red t-shirt, dark red shorts, white socks, red sneakers, brown eyes, warm friendly smile. Nino is a 4-year-old boy with medium skin tone, short messy warm-brown hair, light green t-shirt, dark green shorts, white socks, green sneakers, brown eyes, curious cheerful expression. Sunny neighborhood park with green grass and a paved path. Bright cheerful morning light. 16:9 aspect ratio, landscape composition, zoomed-out view, full-body characters visible, extra empty space suitable for adding educational text later, no speech bubbles, no text, no labels, no watermarks, child-friendly educational illustration style, warm soft colors, simple shapes, friendly expressions, consistent character appearance, ages 4-6.
```

# 6. Page-to-Prompt Mapping

* Exactly **one prompt per story page** (12 total)
* Prompt must depict the primary visual moment of that page's text
* Do not combine multiple pages into one prompt
* Do not preview future plot events not yet stated on that page
* If a page is dialogue-heavy, show the speakers and a relevant prop or gesture

---

# 7. Series Memory in Prompts

When Series Memory provides context:

* Reuse established settings with consistent details (e.g., Sunny Grove Kindergarten reading corner)
* Include recurring Tier 2 characters only when the story includes them
* Subtle callbacks are allowed (same park bench, same kitchen table) when continuity applies
* Do not force callbacks that confuse the current story
* Locked character appearances do not change based on memory — always use bible section 14

Teacher story inputs override memory-based scene choices, not locked character appearances.

---

# 8. Do / Don't

## Do

* Keep prompts copy-ready in one paragraph
* Use concrete visual nouns children can point to (ball, cup, tree, book)
* Show clear facial expressions matching the story mood
* Depict cooperation, curiosity, and gentle problem-solving
* Match weather, time of day, and setting to story text
* Show full-body characters in zoomed-out 16:9 landscape compositions
* Reserve empty space for future educational text overlays
* Use locked character descriptors for every official character on the page

## Don't

* Include text, labels, speech bubbles, or words in the image
* Use watermarks or logos
* Use scary shadows, monsters, injuries, or crying as the focal emotion unless the story resolves it on the same page
* Add characters not present in the page text
* Use photorealistic, cinematic, or adult-targeted styles
* Add brand logos, copyrighted characters, or pop culture figures
* Overcrowd the scene with more than one primary action
* Change locked clothing, hair, or skin tone for official characters
* Crop characters to close-up only when full-body visibility is possible

---

# 9. Setting Reference Prompts

Reuse these setting anchors for consistency:

**Home kitchen:** Cozy kitchen with wooden table, fruit bowl, window with daylight, warm tones

**Sunny Grove Kindergarten:** Bright classroom, colorful cubbies, reading corner rug, bulletin board

**Neighborhood park:** Green grass, wooden bench, small pond with ducks, paved path, trees

**Market Street:** Friendly outdoor market stall with colorful fruit, striped awning, cheerful daylight

Adjust only when story text or teacher Notes specify a different location.

---

# 10. Vocabulary and Learning Goals in Prompts

When a page introduces target vocabulary:

* Show the object or action being learned when possible
* Do not write the vocabulary word as text in the image
* Let the visual reinforce the word the story text already uses
* Use the reserved empty space in the composition for readability — not for rendered text

Example: If the target word is "basket," show Nina holding a woven basket — do not render the word "basket" in the image.

---

# 11. V1 UX Requirements

The product UI should:

* Display the prompt below or beside each story page
* Provide a one-click copy button per prompt
* Keep prompts readable without truncation (wrap or expand)
* Not require teachers to edit prompts for basic usability (editing prompts is optional)

Acceptable V1 shortcut:

* Teachers copy prompts manually into external tools
* No preview image required in the app

---

# 12. Quality Checklist

A prompt is acceptable when:

* [ ] Matches the page text's primary scene
* [ ] Uses correct locked character descriptors from the bible
* [ ] Includes the full locked continuity suffix from section 3
* [ ] Uses 16:9 landscape zoomed-out framing with full-body characters
* [ ] Reserves space for educational text overlays (no text rendered in image)
* [ ] Contains no text-in-image instruction violations
* [ ] Is age-appropriate and classroom-safe
* [ ] Is copy-ready as a single paragraph
* [ ] Does not contradict locked character appearances

---

# 13. Future Scope (Not V1)

The following are explicitly out of V1 scope:

* In-app image generation APIs
* Stored generated images in Supabase
* Image regeneration per page inside the product (beyond prompt text edit)
* Style picker or multiple illustration styles
* Teacher-editable character profiles
* "Edit Characters" button or character management modal
* Character profile persistence in the database

Potential future UI (documentation only — do not implement):

* "Edit Characters" button on story or series settings
* Character management modal with per-character appearance fields
* Saved character profiles that override or extend the locked bible defaults

If added later, update this guide, [character-bible.md](character-bible.md), and [source-of-truth.md](source-of-truth.md) first.

---

# 14. Final Rule

When uncertain about a visual decision:

> Choose the simpler, warmer, more child-friendly composition.

For official characters:

> Use locked bible descriptors and the full continuity suffix. Preserve the original Nina & Nino visual identity.

If illustration behavior would change workflow, outputs, or persistence:

> Update documentation first, then implement.
